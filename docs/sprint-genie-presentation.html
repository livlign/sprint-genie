Architecture Decision Records
Five decisions that shape the build
ADR-001: Tech Stack
React + Express + TypeScript
Single language across the stack. Vite for fast dev, Tailwind for styling. Rejected Next.js (overkill), Python backend (two languages), vanilla JS (hard to maintain).
ADR-002: AI Integration
Claude Messages API + SSE streaming
Backend proxies API calls, streams responses via Server-Sent Events. System prompt adapts to work type. "Done" triggers structured JSON output for the ticket builder.
ADR-003: Jira Integration
REST API v3 + API token auth
No OAuth for v1 — API token is simpler for a single-user local tool. Creates Epic first, then links child tickets. Exponential backoff for rate limits.
ADR-004: Data Persistence
Browser LocalStorage for drafts
Auto-save every 5 seconds. Session list with resume capability. JSON export for backup. ~5MB limit is sufficient. Migration path to SQLite if needed.
ADR-005: Code Browsing
Local filesystem for v1, Bitbucket for v2
During grooming, Claude browses local source code (directory tree, file read, grep search) to ground solution design in reality. .gitignore-aware, path-safe. Bitbucket API in v2 for remote access.
